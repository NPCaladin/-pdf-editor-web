# GitHub Desktop 저장소 경로 확인 및 수정

## 문제: 파일이 보이지 않음

저장소가 다른 폴더에 생성되었을 가능성이 있습니다.

---

## 해결 방법

### 방법 1: 저장소 경로 확인 및 변경

1. **GitHub Desktop에서**
   - 상단 "Current repository" 옆의 드롭다운 클릭
   - "PDF-editor" 선택
   - Repository → Show in Explorer 클릭
   - 파일 탐색기에서 경로 확인

2. **경로가 다르다면**
   - File → Options → Repositories
   - "PDF-editor" 저장소 선택
   - "Remove" 클릭 (파일은 유지)
   - 새 저장소 생성 (아래 참고)

### 방법 2: 새 저장소를 올바른 경로에 생성

1. **GitHub Desktop에서**
   - File → New Repository 클릭

2. **저장소 설정**
   - **Name**: `pdf-editor-web`
   - **Local Path**: `C:\Users\master\Desktop\new_aladin`
     - **중요**: "Choose..." 버튼을 클릭하여 정확한 폴더 선택!
   - Git ignore: None
   - "Create repository" 클릭

3. **확인**
   - 이제 "Changes" 탭에서 파일들이 보여야 합니다

---

## 방법 3: 현재 폴더를 Git 저장소로 초기화

만약 위 방법이 안 되면:

1. **현재 폴더 확인**
   - 파일 탐색기에서 `C:\Users\master\Desktop\new_aladin` 열기
   - 다음 파일들이 있는지 확인:
     - `app.py`
     - `requirements-web.txt`
     - `Procfile`
     - `runtime.txt`
     - `templates/` 폴더
     - `static/` 폴더

2. **GitHub Desktop에서**
   - File → Add Local Repository 클릭
   - `C:\Users\master\Desktop\new_aladin` 선택
   - "Add repository" 클릭

---

## 확인 체크리스트

다음 파일들이 "Changes" 탭에 보여야 합니다:

- ✅ `app.py`
- ✅ `requirements-web.txt`
- ✅ `Procfile`
- ✅ `runtime.txt`

---

## 문제 해결

### 파일이 여전히 안 보임

1. **파일 탐색기에서 확인**
   - `C:\Users\master\Desktop\new_aladin` 폴더 열기
   - 파일들이 실제로 있는지 확인

2. **GitHub Desktop 새로고침**
   - F5 키 누르기
   - 또는 File → Refresh

3. **저장소 다시 추가**
   - File → Add Local Repository
   - 정확한 경로 선택
