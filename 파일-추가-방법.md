# GitHub Desktop에 파일 추가하기

## 현재 상황

- ✅ Git 저장소가 생성됨 (`.git` 폴더 확인됨)
- ❌ GitHub Desktop에서 "PDF-editor"만 보이고 필요한 파일들이 안 보임

---

## 해결 방법

### 방법 1: GitHub Desktop에서 파일 확인

1. **GitHub Desktop 새로고침**
   - F5 키 누르기
   - 또는 File → Refresh

2. **필터 확인**
   - "Changes" 탭의 "Filter" 입력란 확인
   - 필터가 켜져 있으면 비우기

3. **모든 파일 확인**
   - "1 changed file" 체크박스 옆의 화살표 클릭
   - 또는 스크롤하여 다른 파일들 확인

---

### 방법 2: 파일을 수동으로 추가 (필요한 경우)

만약 파일들이 여전히 안 보인다면:

1. **파일 탐색기에서 확인**
   - `C:\Users\master\Desktop\new_aladin` 폴더 열기
   - 다음 파일들이 있는지 확인:
     - `app.py`
     - `requirements-web.txt`
     - `Procfile`
     - `runtime.txt`
     - `templates/index.html`
     - `static/app.js`
     - `static/style.css`

2. **GitHub Desktop에서**
   - "Changes" 탭 확인
   - 파일들이 보이지 않으면:
     - Repository → Show in Explorer 클릭
     - 파일 탐색기에서 파일 확인

---

### 방법 3: .gitignore 확인

`.gitignore` 파일이 너무 많은 파일을 제외하고 있을 수 있습니다.

1. **파일 탐색기에서**
   - `.gitignore` 파일 열기
   - 내용 확인

2. **필요한 파일이 제외되어 있다면**
   - `.gitignore` 파일 수정
   - 또는 GitHub Desktop에서 "Show ignored files" 옵션 확인

---

## PDF-editor 폴더 처리

"PDF-editor" 폴더는 필요 없습니다:

1. **GitHub Desktop에서**
   - "PDF-editor" 체크 해제
   - 또는 무시

2. **파일 탐색기에서** (선택사항)
   - `PDF-editor` 폴더 삭제 가능

---

## 확인해야 할 파일들

다음 파일들이 GitHub Desktop의 "Changes" 탭에 보여야 합니다:

**필수 파일:**
- ✅ `app.py`
- ✅ `requirements-web.txt`
- ✅ `Procfile`
- ✅ `runtime.txt`
- ✅ `.gitignore`
- ✅ `templates/index.html`
- ✅ `static/app.js`
- ✅ `static/style.css`

**제외할 파일 (.gitignore에 의해 자동 제외):**
- ❌ `build/` 폴더
- ❌ `dist/` 폴더
- ❌ `temp/` 폴더
- ❌ `main.py` (데스크탑 앱용, 웹앱에서는 불필요)

---

## 다음 단계

파일들이 보이면:

1. **PDF-editor 체크 해제** (필요 없음)

2. **필요한 파일들만 선택**
   - `app.py`
   - `requirements-web.txt`
   - `Procfile`
   - `runtime.txt`
   - `templates/` 폴더
   - `static/` 폴더

3. **커밋**
   - "Summary": `Initial commit: PDF Editor Web App`
   - "Commit to main" 클릭

4. **GitHub에 푸시**
   - "Publish repository" 클릭

---

## 문제 해결

### 파일이 여전히 안 보임

1. **GitHub Desktop 완전히 종료 후 재시작**

2. **저장소 다시 추가**
   - File → Add Local Repository
   - `C:\Users\master\Desktop\new_aladin` 선택

3. **파일 탐색기에서 직접 확인**
   - 파일들이 실제로 있는지 확인
